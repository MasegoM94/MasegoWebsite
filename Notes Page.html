FBProphet


        web

12/04/2020

" Prophet is a forecasting procedure implemented in R and Python . It is fast and and provides completely automated forecasts that can be tuned by hand by data scientists and analysts", a quote I sourced from the Prophet website. Prophet doesn't solve all forecasting problems, they do mention on their Facebook Research page on which kinds of forecasting scenarios that their product/model is best suited for. One scenario they mention is for "daily or weekly observations where there is a few months of history", and this is how I got introduced to fbProphet. I am currently doing a course on Udemy titled Machine Learning Practical Workout | 8-Real-World Projects and two of the projects used the fbProphet package. I do not know how much I can share so I will generalize. The one case was predicting a rate and the other is predicting a future price of a commodity. From what I saw, the package ws fairly straightforward to use. However, what I wish is that the instructor would explain more of what is happening under the hood of the different commands. One thing I worry about for this package is that one's data needs to be transformed before it is entered into the mechanism. I would like to try it on a big dataset using pyspark. On their page they mention that scaling is not really a problem for time series data, I am guessing because only two variables are required (from what I have been introduced to). From what I see, it would not be ideal for a multi-variable time series problem. The Prophet procedure is an additive regression model (which basically is a linear model where the change in one variable doesn't affect the change in another). The procedure will then output future predictions (dependent on the users specifications), trend line with accompanying uncertainty band, weekly and yearly seasonality plots depending on the type of data entered. This package can be quite powerful if fully understood by the user.

What the hell are Epochs anyway?


        web

27/04/2020

I have to admit, I had been struggling with the idea of what an epoch (in neural networks) actually is (Yes, I know I claim I am a Data Scientist and what not but some concepts are not so straightforward). With this Udemy course I am doing I came across epochs in neural networks again . It was probably the best eplanation I have came across. So the instructor described it as each time the weights in the artificial neural networks update. Ah of course that makes sense. I came across another definition which made the concept even clearer on DeepAI.org. They say that epochs are not iterations which is honestly what I have been mistaking them as. More interestingly, they gave a workable example of how an epoch and iteration can interpreted in the case of a reinforcement learning example. To sum it up an epoch would be one completion of task whilst an iteration is an action within a task. I hope that makes sense. I also came across another definition that could ease the understanding of an epoch. This one is in terms of batch sizes. A batch size is the number of data points you want to enter into a model at a time. This is a hyperparameter that is usually considered due to one's machine's computing power. An epoch is then when the entire training data has gone through the neural network once. How long one epoch could take is dependent on the batch size. Look at us learning and stuff.

A visual application of Image Kernels


        web

08/05/2020

For one of the projects available on the Udemy course that I am currently doing, I was introduced to this Image Kernels website. An Image Kernel is a matrix that transforms an image to a specific effect the user would like to apply to that image for example blurring the image. The project had to do with images and using convolutional neural networks to classify these images into classes. In the case of convolutional neural networks, image kernels are used for feature extraction when building the network. This website illustrates how the transformation is done on the image according to whichever effect the user would like to see happen to the image. They illustrate this transformation visually and mathematically. The concept is genius and I truly appreciate the developers of this website for sharing it. They also allow the user to upload an image or video and then apply these kernels to them.

LeNet-5 Architecture?


        menu_book

18/05/2020

For this post, I have a question to pose but I will leave that for the end. With the Udemy course that I am doing, there is a project that introduces the use of LeNet-5 structure Convolutional Neural Networks (CNNs). The LeNet CNN has a very specific architecture that was introduction by LeCun et al. in 1998. The basic structure, as seen in the image, has a convolutional -, subsampling -, convolutional -, subsampling - and then convolutional layer used to flatten the data to two fully-connected layers that eventually feels to an output layer. Note this is just one of many architectures that can be used for image classifications. There are also some dimension calculations that are required between the first 5 layers of the network. In the original paper, they used average pooling in the subsampling layer. What I found confusing, which has lead to my question, is can we still call the structure/architecture LeNet if we use a different subsampling method, like maxpooling? I came across a LeNet tutorial that used the maxpooling method instead of average pooling. It was the only LeNet explanation I found that was different, which confused me (thanks DeepLearning.net). This brings me back to my question. What features of the structure make it LeNet? I don't mind settling with just calling it a variation of the original structure but at what point of change from the original structure do we stop calling it a variation?

Confusion Matrix and Classification metrics


        web

18/05/2020

In my line of work, evaluating models is very essential when settling on a model. However, in corporate I have noticed that we have a tendency at always looking at the same metrics which I attribute to business having to understand the evaluation versus just the technical person. With this post I just wanted to note the metrics available for the classification problems. Before we get to the metrics, I need to bring attention to the confusion matrix. I have added an example alongside. It is a table that documents the predicted classes against the actual classes and allows us to see how and where there were correct and incorrect predicted classifications. This matrix is important in calculating other metrics that will be discussed.

Accuracy is the most commonly used metric. It gives us the percentage of times the model predicted the correct class. This could be misleading if there is an imbalance in the classes in the data.

Misclassification / Error rate gives us the percentage of times the model predicted incorrectly [1 - Accuracy].

Precision gives us percentage of times when the model predicted a certain class, and how often it was right. This is could be like looking at accuracy of prediction of a particular class. When the model predicted TRUE how often was it right.
TP /(TP + FN)

Recall/Sensitivity is the percentage of times a particular class was predicted correctly. When the actual value is TRUE how often did the model correctly predict the value to be TRUE.
TP /(TP + FP)

Because we would like a model to be high for both precision and recall and therefore there is another metric that is created as a result of considering both metrics, called an F-measure (F1-Score). It is the Harmonic mean of the two measures.
(2 X Precision X Recall)/(Precision+Recall)

Specificity is like if we were looking at recall but for actual FALSE. When the actual value is FALSE how often did the model correctly predict the value to be FALSE.
TN /(TN + FP)

This medium article tells us when to use which metric. The article also introduces the concept of a weighted F1 score as we don't always weight the precision and recall the same
(1+ β²)(Precision X Recall) /( ( β² X Precision ) + Recall)
where β times as importance to Recall as Precision

This same article also has 3 more metrics, that haven't been discussed here, i.e. Logloss/Binary Crossentropy, Categorical Crossentropy and Area under the Curve (AUC). Also note that crossentropy's are susceptible to class imbalance and it is suggested that one uses a weighted approach.
